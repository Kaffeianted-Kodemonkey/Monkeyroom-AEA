A11y Evaluation Audit (AEA)

All Evaluation Audits are conducted using the following process:

  1. Launch the AEART to recored test results
  2. Test WCAG Criteria using A11y tools, screenreads, & assitive technologies

Make sure to report all accessibility issues in the AEA reporting tool.

Submit both the json and html files to AEA-Complete.

Hint:

We know it's unrealistic to conduct a manual evaluation on every page of a large site. We suggest reviewing these key pages to start:

  • Site templates
  • Interactive forms
  • Dynamic content pages
  • Pages with dialog modals and alert notifications
  • Key entry and exit pages (including account login and recovery pages)
  • Help and assistance pages

AEA Reporting Tool

The A11y Evauation Audit reporting tool is used to record the results from a manuel evaluation audit.

AEA Report

  1. Select Generate Report button below.
  2. Select Choose File and load JSON file with test results.
  3. Select Open.
  4. Select Load Results button.
  5. View the Test Results.
  6. Select the Save as HTML button at the bottom or use Alt+S to generate a printer-friendly version of the report and save JSON file for future use.
