Welcome to the {{ page.title}} Process!

At Kaffeinated Kodemonkey, we uphold the highest standards of accessibility, ensuring that all our products meet ADA Compliance requirements. Our commitment to accessibility is reflected in our rigorous Accessibility Evaluation Audit (AEA) process, where we utilize a range of A11y tools, screen readers, and assistive technologies.

As an A11y Developer, Lead Moneky or Kodemonkey, your role is pivotal in guaranteeing that the product not only meets but exceed accessibility standards. Following the A11y Evaluation Audit Guidelines, you will assess the product using screen readers, mobile devices, assistive technologies, WAVE, and any other specified tools outlined in the project scope.

Before embarking on any project, it's essential to thoroughly review the project scope. This will provide clarity on the required level of compliance, whether it's WCAG/508, WCAG only, AA, or AAA. Additionally, take note of any unique testing methods suggested or required for the project.

Getting started with the AEA process:

Understanding the challenges of manually evaluating every page, especially on large sites, we have streamlined the AEA process. We will provide a list of URLs required for review within the Project Space, along with the following critical areas for assessment:

  • Site templates
  • Interactive forms
  • Dynamic content pages
  • Pages with dialog modals and alert notifications
  • Key entry and exit pages (including account login and recovery pages)
  • Any pages that process payments
  • Help and assistance pages

Instructions

All AEA will be conducted using the DHS Trusted Tester Process outlined below.

Locate the Test Results Form
Scroll down to find the Enter Test Results form. It is only visible after a valid JSON file is loaded
Enter the Test Results
Each row in the Test Results Form includes:
  • A. Issue Number
  • B. Test Name
  • C. Test ID
  • D. Test Condition
  • E. Test Result - Use the dropdown to select
    • No Selection
    • Pass
    • Fail
    • Does Not Apply
  • F. Browser - If the browser used for this test result is different it can be noted here. When applicable, select the check box to change the browser. Use the dropdown to select the version of the browser used for the test.
  • G. Tester Comments - The form field has a character limit and is used for any comments about the test.
  • H. Location/Screen - The form field has a character limit and is used to provide details needed to locate the defect on the web page.
  • I. Screenshot - After a test result is added, a ‘Choose File’ button will appear to add images to the corresponding test result. To add a screenshot, select the ‘Choose File’ button, select the image file (PNG, JPEG, or GIF), and select Open.
  • J. Global Issue - Use the dropdown to select
    • None
    • Yes
    • No
  • K. Go to Parent/Add Child Issue - The Child Issue is a separate issue added that is associated to the same Test ID, the parent.
