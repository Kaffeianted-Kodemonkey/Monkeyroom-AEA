At Kaffeinated Kodemonkey, we prioritize accessibility and believe that all products should meet the highest standards of ADA Compliance. To achieve this, we follow a rigorous process that involves the use of A11y tools, screen readers, and various assistive technologies aligned with our Accessibility Evaluation Audit (AEA) process.

As an A11y Developer or Kodemonkey, your role is crucial in ensuring that products meet or exceed accessibility standards. You will evaluate the product following the A11y Evaluation Audit Guidlines, using screen readers, mobile devices, assistive technologies, WAVE, and any specified technologies outlined in the project scope.

Before diving into any projects, it's crucial to thoroughly review the project scope. This will provide clarity on the level of compliance required, whether it's WCAG/508, WCAG only, AA, or AAA. Additionally, take note of any required or suggested testing methods unique to the project.

Getting started with the AEA process:

To streamline the AEA process, we understand the impracticality of conducting a manual evaluation on every page of a large site. Therefore, within the project scope, you'll receive an Audit Report outlining the current status of the project. It's suggested to kickstart the AEA reporting by verifying these results.

For most sites utilizing templates/themes, key items to review include:

  • Site templates
  • Interactive forms
  • Dynamic content pages
  • Pages with dialog modals and alert notifications
  • Key entry and exit pages (including account login and recovery pages)
  • Any pages that process payments
  • Help and assistance pages

Instructions

All Evaluation Audits are conducted using the following process:

  1. Launch A11y Evaluation Audit to record test results
  2. Selecta previous JSON File or load a new JSON file: First time use Trusted Tester v.5.1.1 (for web) or Trusted Tester v.4 (for software) (TT5.1.1 or TT4)
  3. Select Load File button.
  4. Referande the Project Scope to complete each form and the required fields.
  5. Add/Edit Test Results as needed
  6. Save the file as-is in JSON format
  7. Upon completion submit the JSON file to The A11y Dev Team.

Remember, the Project Scope serves as a comprehensive guide, outlining any special instructions and providing essential information required to complete the AEA Report successfully.

  • C. Test ID
  • D. Test Condition

    • No Selection
    • Pass
    • Fail
